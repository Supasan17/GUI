The aim 01: Define all strict TypeScript interfaces for the DummyJSON API responses without using any "any" types
Actual prompt: I need TypeScript interfaces for the DummyJSON products API. The product object has: id, title, description, category, price, discountPercentage, rating, stock, tags (array), brand, sku, weight, dimensions (width/height/depth), warrantyInformation, shippingInformation, availabilityStatus, reviews (array with rating/comment/date/reviewerName/reviewerEmail), returnPolicy, minimumOrderQuantity, meta (createdAt/updatedAt/barcode/qrCode), images (string array), thumbnail. Also create interfaces for the paginated response wrapper, login credentials, and the JWT auth response. Make everything strictly typed with no "any".

The aim 02: Set up a Pinia store for authentication that uses the DummyJSON /auth/login endpoint and persists the JWT to localStorage
Actual prompt: Create a Pinia store in Vue 3 with the Composition API style (defineStore with setup function) for authentication. It should: call POST https://dummyjson.com/auth/login with username and password, store the returned JWT token and user object in both the store's reactive state and localStorage, expose isLoggedIn (computed), a login() function that returns a boolean indicating success or failure, and a logout() function that clears both state and localStorage. Handle loading and error states. Rehydrate state from localStorage on store initialization.

The aim 03: Create a persistent shopping cart Pinia store with quantity controls and discount calculations
Actual prompt: Build a Pinia cart store for a Vue 3 TypeScript e-commerce app. It should: store CartItem[] (each with a Product and quantity), persist to localStorage after every mutation, expose a computed "summary" that returns subtotal (full price × qty), discount (savings from discountPercentage), total (subtotal minus discount), and itemCount. Include addToCart (prevent exceeding product.stock), removeFromCart, updateQuantity (auto-remove if qty goes to 0), clearCart, and helpers isInCart(id) and getQuantity(id). No "any" types.

The aim 04: Build a FilterBar component that handles search, category selection, sort, and price range — all via emitted events
Actual prompt: Create a Vue 3 TypeScript component called FilterBar. It receives: a "filters" prop of type ProductFilters (search: string, category: string, sortBy: SortOption, minPrice: number, maxPrice: number), a "categories" string array, and a "totalResults" number. It should NOT mutate props directly — instead emit "update:filters" with the full updated filters object whenever anything changes. Include: a text input for search, a set of badge-style toggle buttons for category (including an "All" option), a select dropdown for sort (default/price-asc/price-desc/rating/name), a range slider for max price showing the current value, and a "Reset filters" button. Use Tailwind for styling.
